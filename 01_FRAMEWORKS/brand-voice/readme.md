# Brand Voice System — Overview

This folder contains the complete Brand Voice System used within the DGA Platform.  
It defines how client voice, tone, and messaging preferences are captured, documented, and applied across all content and systems.

The Brand Voice System ensures:
- Consistent tone and messaging across all platforms  
- Authentic client representation  
- Stronger GEO signals  
- Alignment across OCGF, audits, onboarding, and REA engagement  
- Repeatable workflows for all DGA subscribers  

## Folder Contents

### **1. framework.md**
The master document explaining the entire Brand Voice Framework, including purpose, outcomes, workflows, and integration across DGA systems.

### **2. interview-script.md**
The structured script used during onboarding interviews to capture a client’s authentic voice and communication style.

### **3. blueprint-template.md**
The standardized template used to create the Brand Voice Blueprint (BVB) for each MP or REA client.

### **4. ai-wrapper.md**
A reusable AI system prompt ensuring all ChatGPT or automation-generated content follows the client’s Brand Voice Blueprint exactly.

### **5. notion-implementation.md**
Instructions for how the Brand Voice System is stored, managed, and used inside the DGA Notion workspace, including CRM integration.

## Usage Workflow

1. Conduct the Brand Voice Interview → using **interview-script.md**  
2. Complete the Brand Voice Blueprint → using **blueprint-template.md**  
3. Store in Notion and link to CRM → see **notion-implementation.md**  
4. Use BVB for all content creation → supported by **ai-wrapper.md**  
5. Reference **framework.md** for operational or strategic consistency
