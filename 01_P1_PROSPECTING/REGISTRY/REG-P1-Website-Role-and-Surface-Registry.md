## 1. Authority & Scope

This Registry defines the **canonical classification, role, and permitted surfaces** of the DGA website.

It is:
- authoritative
- non-executing
- logic-free
- binding on all systems, documents, and humans

If a system, Execution Doctrine, SOP, or page behavior conflicts with this Registry, **this Registry prevails**.

---

## 2. Registry Rules (Non-Negotiable)

- All fields defined here are **immutable**
- No implicit interpretation is permitted
- If a surface or behavior is not explicitly allowed, it is prohibited
- The website may not assume ownership outside P1
- Registry compliance is required before deployment

## 4. Allowed Surface Types

The following surface types are **explicitly permitted** on the DGA website:

| Surface Type | Description |
|-------------|-------------|
| belief_articulation | Declarative expression of DGA beliefs and worldview |
| system_framing | High-level explanation of how DGA operates without tactics |
| boundary_setting | Explicit statements of what DGA does and does not do |
| expectation_setting | Clear articulation of assumptions required to proceed |
| self_selection_guidance | Language that enables prospects to opt in or out |

No other surface types are allowed.

---

## 5. Prohibited Surface Types

The following surface types are **explicitly prohibited** on the DGA website:

| Surface Type | Description |
|-------------|-------------|
| sales_pages | Any page designed to persuade or convert |
| pricing_pages | Display of pricing, plans, or comparisons |
| calls_to_action | Buttons, forms, or language designed to drive conversion |
| intake_triggers | Any mechanism that initiates P2 Intake |
| audit_previews | Audit samples, diagnostics, or implied evaluations |
| dashboards | Any subscriber-facing or system-facing interface |
| testimonials | Social proof or outcome validation |
| case_studies | Performance narratives or success stories |
| optimization_language | Language implying improvement, growth, or results |

Presence of any prohibited surface invalidates compliance.

## 6. Read / Write / Execution Constraints

| Capability | State |
|-----------|-------|
| read_access | Enabled (Public) |
| write_access | Prohibited |
| execution_logic | Prohibited |
| automation_triggers | Prohibited |
| system_state_changes | Prohibited |

The website must remain **structurally inert**.

## 7. Pillar Ownership Declaration

- The DGA website is owned exclusively by **P1 — Prospecting**
- No other pillar may project functionality, authority, or execution onto the website
- Any downstream reference must treat the website as **non-authoritative**

Ownership transfer is not permitted.

## 8. Downstream Consumption Rules

This Registry must be consumed by:

- ED–P1–Website Governance Rules
- Any system implementing the DGA website
- Any human-authored page or content placed on the website

Consumption must be literal and non-interpretive.
